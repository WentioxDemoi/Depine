  2. Configure my project
    1. Switch to Editor Mode

      2. Everytime you want to edit or configure something, simply switch to Editor Mode, at the top of the window.

    2. Select the project folder

      3. First, select your project's folder in your file explorer, like you would in your IDE. Simply click on the "Add" button.

      ðŸ’¡The folder has to be a Git folder.

      This will create a new empty Pinefile. This is where all the project dependencies will be written. Also, it will add a banner at the top of your README.md.

      This indicates to new developpers that they can use Depine to deploy this project.

      ðŸ’¡This is not mandatory, but we highly recommand you to leave it here, or at least mention it in your README.md.
    3. Select the dependencies

      4. Select the dependencies you project needs. Simply search and add them. The dependencies are separated into four distinct categories :

      • Languages - pretty self-explanatory..?

        e.g. Java @11

      • Utilities - it could be package managers or diverse software.

        e.g. Maven @3.81, Docker @latest

      • IDEs

        e.g. IntelliJ IDEA @latest

      • Scripts- the commands Depine needs to run to complete installation.

        e.g. mvn clean install, mvn spring-boot:run

      What if I don't find the dependency ?

      We got you covered : if the search doesn't find a dependency, you can upload the installer yourself. Your team will get a warning when deploying though, with your name.

      If the dependency is frequently added, we'll make sure to add it to our database : the warning will also disapear.

    4. Fine tune the dependencies

      5. If needed, you can configure the dependencies configuration. That could be an IDE plugin, some Maven auth settingsâ€¦

      Simply click the edit button on a dependency.

    5. Save your work

      6. Don't forget to save your changes. Simply hit Ctrl+S / Cmd+S or click on the "Save" button in the top-right corner.

    6. Push the Pinefile

      7. After saving your changes, it's important to push the Pinefile (and the README.md) to your Git repository. This will allow your teammates to access the latest project configurations. Simply use your usual Git tool to add, commit, and push the Pinefile to your main branch.
